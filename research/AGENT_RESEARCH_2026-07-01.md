Direct inspection of the live repo this cycle shows the "primary language: HTML / notebook-heavy" flag carried in `AGENT_TASK_QUEUE.md` priority #2 is now stale: the repo contains zero `.ipynb` files and is already a Python-first, "production ready" pipeline (`config.py`, `feature_engineering.py`, `pipeline.py`, `model_comparison.py`, `streamlit_app.py`) with a Random Forest model in production (F1=0.432, ROC-AUC=0.760, Recall=91.6% on the 2024 held-out set). ## Engineering Insights
- Repo status correction: this cycle cloned the live repo directly (shallow clone, header-auth) rather than relying on the stale GitHub-language-detection flag — confirms zero notebooks, all logic already in top-level `.py` modules. Future cycles should stop treating "notebook refactor" as an open task unless re-verified.
- FastF1 caching (`fastf1.Cache.enable_cache()`) remains the top engineering tip across all surveyed sources — repo's `config.py`/`load_real_data.py` should confirm this is enabled before any new data-pull scripts are added, to avoid API rate-limit issues as the pipeline grows.
- The Hetang2403 optimizer's feature-importance ranking (stint % > compound > safety car > round) matches this repo's own documented feature set almost exactly — good external validation, worth a one-line README citation.
- SOTA papers increasingly pair the predictive/RL model with an explainability layer (decision-tree surrogate, SHAP) rather than shipping a black-box model — this repo's Streamlit dashboard currently has feature-importance bars but no per-prediction (per-lap) attribution, which is the gap to close.
- Every RL-based competitor repo keeps the race/tire simulator (`env/`) architecturally separate from the policy (`agent/`) — worth keeping in mind if/when Austin decides to add an RL track alongside the existing supervised pipeline.
